# Copernicoaches

Repositorio de trabajo del **workshop de coaches de adopción de IA en Copernico**.

Este repositorio contiene los materiales, ejercicios y proyectos utilizados durante el workshop. Cada día del programa tiene su propia carpeta con el código base, instrucciones y artefactos producidos por los equipos.

El objetivo del taller no es enseñar el uso básico de herramientas de IA, sino preparar a los participantes para trabajar en un modelo de **desarrollo guiado por especificaciones**, con uso estructurado de **GitHub Copilot** y especial atención a la **ingeniería de contexto**.

---

# Estructura del repositorio

```

/dia1
/dia2
/dia3
...
/dia10

```

Cada carpeta corresponde a un día del workshop y contiene:

- código base o proyecto inicial
- instrucciones del ejercicio
- materiales de apoyo
- artefactos generados durante el trabajo

Los equipos trabajarán dentro de estas carpetas o en subcarpetas propias.

---

# Forma de trabajo

El workshop se organiza como una secuencia de **misiones diarias**.

Cada día sigue la misma lógica:

1. comprender el problema o contexto
2. estructurar la información
3. producir artefactos claros (documentación, diagramas, specs)
4. utilizar IA dentro de un flujo de trabajo controlado
5. entregar un resultado visible al final del día

El repositorio NO sirve como espacio común de trabajo y registro del progreso! **Cada equipo tiene que crear su proprio proyecto** donde tiene que subir los cambios. 

---

# Uso de Git

Cada equipo debe:

1. clonar el repositorio
2. trabajar en su carpeta correspondiente
3. realizar commits frecuentes
4. subir los cambios a su proprio repositorio(!)


Durante el workshop se espera que cada repositorio refleje:

- el progreso de los equipos
- las decisiones tomadas
- los artefactos producidos

---

# Uso de GitHub Copilot

GitHub Copilot se utilizará como **apoyo al trabajo**, no como sustituto del razonamiento.

En particular se espera que los equipos utilicen Copilot para:

- analizar código existente
- explicar errores o comportamientos
- proponer soluciones técnicas
- ayudar en la implementación

Antes de implementar cambios relevantes, se recomienda producir una **pequeña especificación** que describa qué se quiere construir.

---

# Artefactos esperados durante el workshop

A lo largo del programa los equipos producirán diferentes tipos de artefactos:

- documentación en Markdown
- diagramas (Mermaid)
- especificaciones técnicas
- prompts estructurados
- implementaciones parciales
- pruebas y revisiones

Estos artefactos deben quedar registrados en el repositorio.

---

# Resultado esperado

Al final del workshop los participantes deberán ser capaces de:

- analizar contexto técnico complejo
- transformarlo en artefactos comprensibles
- escribir especificaciones útiles
- utilizar IA dentro de un flujo estructurado de trabajo
- acompañar a otros equipos en la adopción de este modelo

El repositorio **Copernicoaches** servirá como referencia del trabajo realizado durante el programa.
